# Quest Task Manager v1.2.14

**Поколение 1.2** — эволюция 1, второе поколение функционала.  
Базовый релиз поколения: **v1.2.0** (вкладка «Лог»).  
**v1.2.14** — накопительный стабильный релиз: все исправления и доработки с момента 1.2.0 без публикации промежуточных сборок.

---

## Списки приложений

- Фильтр **«Пользовательские»** по умолчанию; chip совпадает с реальным фильтром списка.
- Корректная классификация **пользовательские / системные**: системные процессы из dumpsys больше не попадают в «Пользовательские» (UID, префиксы Meta/Oculus/Android).
- Список **«Запущенные»**: только установленные APK; нативные процессы (`media.*`, `hidl.*` и т.п.) убраны из обычных фильтров.
- Расширенный сбор запущенных: `ps` + `dumpsys activity processes` + `meminfo` (без «призраков» из recents).
- **Тап по карточке** приложения на обеих вкладках → детали в Android Settings (иконка, название, память; переключатели на вкладке «Приложения» не перехватывают клик).

---

## Закрытие приложений (kill)

- Исправлен **краш Shizuku** при `am force-stop` (`process hasn't exited` / `exitValue()`).
- Корректный **отчёт** после закрытия: нет ложного «закрыто 0 / не удалось 1» при успешном `force-stop`.
- Успех kill определяется по **`am force-stop`**, без ложных ошибок из‑за фоновых `*.service.*` в `ps`.
- Цепочка завершения: `am force-stop` → `am kill` → при необходимости `kill -9` по PID; учёт защищённых и неудачных.

---

## Настройки Android

- Кнопка и тап по приложению открывают **главную страницу** Settings, а не «Система» / режим разработчика / «Все приложения».
- Рабочий запуск на Quest через **`DeepLinkHomepageActivity`** (exported); неэкспортируемая `SettingsHomepageActivity` не используется.
- Shell-запуск с корректным экранированием **`$`** в именах activity; приоритет shell над intent где нужно.
- Если **`com.android.settings`** не установлен — понятный **диалог** со ссылками (QGO, XR Native Android Settings, Hidden Settings); обновлён README.

## Уведомление и фоновая очистка

- Уведомление **включено по умолчанию**; сервис синхронизируется при старте и возврате в приложение.

---

## Скорость работы и кэш списков

- **Первый запуск:** обе вкладки («Запущенные» и «Приложения») загружаются **параллельно**; каждая показывает список **сразу по готовности**, не дожидаясь второй.
- **Повторный запуск:** списки подгружаются из **кэша** последнего сеанса — без долгого ожидания Shizuku.
- После **kill** автообновляется только вкладка **«Запущенные»**; «Приложения» — по кнопке **«Обновить»**.

---

## RAM на вкладке «Запущенные»

- Между фильтрами и кнопкой **«Обновить»** — **свободная / общая RAM** (тот же формат, что в уведомлении, например `2.1/5.8 GB`).
- Обновляется вместе со списком: старт, «Обновить», после kill.

## Установка

Требуется **Shizuku** на Quest. APK: `QTaskMgr-v1.2.14-release.apk`.

Рекомендуется обновление с **v1.2.0** или любой более ранней версии 1.x.
